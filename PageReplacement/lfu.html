Least Frequently Used Page Replacement

In LFU (Least Frequently Used) algorithm the page which has the smallest count is going to be replaced. The reason for this selection is that a mostly used page should have a large reference count. This algorithm suffers from the situation in which a page is used heavily during the staring phase of aprocess, but then is never used again. Since it was used heavily, it has a large frequency (count) and remains in memory even through it is no longer needed.
